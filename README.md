# ECOSTRESS vs CRW Thermal Analysis — Belize 2023

Code repository for the paper. Contains all data processing, analysis, and figure-generation scripts used to compare ECOSTRESS land-surface temperature (70 m) against NOAA CoralWatch (CRW) sea surface temperature (5 km) across Belizean Marine Protected Areas during the 2023 coral bleaching event.

## Data Requirements

The following input files are required but not distributed with this repository due to file size. Instructions for obtaining each are given below.

**Place all files in a `data/` directory** before running notebooks.

| File | Source |
|------|--------|
| `dhw_5km.nc` | NOAA CRW 5 km SST/DHW product — use `download_dhw_data.sh` or the [CRW data portal](https://coralreefwatch.noaa.gov/product/5km/index.php) |
| `ct5km_climatology_v3.1.nc` | NOAA CRW climatology — same portal as above |
| `ECOSTRESS_LST_Mean_Cloudmasked_Composite_May-Nov23.tif` | Generated by running the ECOSTRESS processing pipeline (notebooks 1–7) |
| `ecostress_monthly/masked_ECOSTRESS_LST_Median_*.tif` | Generated by the processing pipeline |
| `Belize_MPAs_Reprojected.shp` (+ sidecar files) | Belize MPA shapefile — available from [Protected Planet (BLZ)](https://www.protectedplanet.net/country/BLZ) or the [Belize Fisheries Department](https://www.fisheries.gov.bz/) |
| `merged_bleaching_data.csv` | Global bleaching survey database — [BCO-DMO dataset 773466](https://www.bco-dmo.org/dataset/773466) |
| `bleaching_confusion_labeled.csv` | Derived from `merged_bleaching_data.csv` by running `crw_dhw_matchup.ipynb` |
| `mean_bleaching_surveys_per_location_Jun-Dec2023.csv` | Derived from bleaching survey data, aggregated to site-level monthly means |

## Processing Pipeline

### Step 1 — Download CRW Data

```bash
bash download_dhw_data.sh    # degree heating weeks (1985–2024)
bash download_sst_data.sh    # SST (1985–2024)
bash download_ssta_data.sh   # SST anomalies (1985–2024)
```

Set the `DEST_PATH` variable in each script to your local data directory before running.

### Step 2 — ECOSTRESS Processing (run in order)

Raw ECOSTRESS L2T scenes are downloaded from [NASA AppEEARS](https://appeears.earthdatacloud.nasa.gov/). Then run the following notebooks in sequence:

1. **`ECOSTRESS_QC.ipynb`** — Pairs each LST scene with its QC band, converts raw counts to °C, masks pixels with QC flag > 1 (cloud / not produced), and skips scenes where more than 90% of pixels are masked.

2. **`ECOSTRESS_Geolocation.ipynb`** — Applies geolocation correction to remove scan-line artifacts and spatially align scenes to a common water-mask reference using GDAL and scikit-image morphological operations.

3. **`Rename_time_classification_ECOSTRESS_files.ipynb`** — Parses the UTC acquisition timestamp from each ECOSTRESS filename and renames files with a local-time suffix (UTC−6 for Belize). Required for correct temporal matching downstream.

4. **`ECOSTRESS_land_mask.ipynb`** — Applies the GSHHG full-resolution land mask to retain only ocean/reef pixels. Land polygons are reprojected on first use to match each raster's CRS.

5. **`ECOSTRESS_manual_cloud_removal.ipynb`** — Interactive inspection tool (static + Bokeh hover map) for identifying and removing residual cloud-contaminated pixels below a user-defined temperature threshold.

6. **`ECOSTRESS_Monthly_Seasonal_Composites.ipynb`** — Builds pixel-wise monthly and seasonal (MJJ, ASO, NDJ, FMA) composites from the cleaned scenes.

7. **`ECOSTRESS_Select_Composites.ipynb`** — Produces max, median, mean, and valid-pixel-count composites from any subset of scenes.

### Step 3 — CRW Processing

**`CRW_composites.ipynb`** — Subsets the CRW SST NetCDF to the Belize study region and time window, then exports a mean composite and valid-pixel-count GeoTIFF.

### Step 4 — Survey Data Matchup

**`crw_dhw_matchup.ipynb`** — Spatially and temporally matches each coral survey record in `merged_bleaching_data.csv` to the nearest-neighbor DHW value from the corresponding daily CRW NetCDF file. Runs in parallel across all CPU cores. Outputs per-year CSVs and a combined file spanning 1985–2020.

### Step 5 — Analysis and Figures

**`Global_bleaching_data_and_significance_test.ipynb`** — Fits a mixed-effects model (site and year as random effects) and runs Mann-Whitney U pairwise tests with Benjamini-Hochberg correction to evaluate the relationship between DHW alert level and percent bleaching globally.

**`MMM_Analysis.ipynb`** — Samples ECOSTRESS LST and CRW SST at each survey site, computes MPA-specific MMM anomalies, and produces the styled per-MPA comparison table and heatwave severity maps (Figure 3).

**`Belize_MPA_vs_max_LST_and_SST.ipynb`** — Scatter plots of bleaching vs monthly maximum temperature for two focal MPAs across both sensors, with Pearson R² and p-value annotations. Produces Figures 5 and S2.

**`Timeseries_by_MPA.ipynb`** — Generates one time series figure per MPA showing daily CRW SST, DHW heat-stress shading (NOAA alert palette), monthly ECOSTRESS LST, climatological SST, MMM threshold, and bleaching survey boxplots (Figure 6).

## Dependencies

All notebooks were developed in Python 3.10. Install dependencies with:

```bash
pip install numpy pandas xarray rioxarray geopandas rasterio scipy \
            matplotlib seaborn cartopy scikit-image statsmodels joblib \
            tqdm netCDF4 gdal shapely
```

For interactive cloud inspection in `ECOSTRESS_manual_cloud_removal.ipynb`:

```bash
pip install hvplot holoviews panel bokeh datashader jupyter_bokeh
```
